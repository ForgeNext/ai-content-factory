# Purpose

Blueprintを構成する各文書の責務を明確にし、
重複・漏れ・責務の混在を防ぐ。

Blueprintは会社全体の設計図であり、
各文書は定められた責務だけを持つ。

# Knowledge

KnowledgeはBlueprintではない。

Knowledgeは会社資産を循環・改善する仕組みである。

Knowledgeから必要な改善のみBlueprintへ反映する。

Blueprintは実証済み設計を保持する。

---

# Responsibility Rule

内容を追加する前に、

「その責務は本当にその文書か」

を確認する。

責務が異なる場合は追加しない。

---

# Update Rule

Blueprintは、

Implementation

↓

Operation

↓

Knowledge

↓

Audit

↓

Blueprint Update

の順で更新する。

思いつきだけでは更新しない。
